## 1. 产品概述

### 1.1 产品名称

**DataForge Lite** — AI 语音训练数据生成工具

### 1.2 产品定位

面向 AI 语音合成（TTS）模型训练的数据预处理工具，将原始音频自动化处理为标准化的训练数据集。

### 1.3 目标用户

- AI 语音研究者 / 开发者  
- TTS 模型训练工程师  
- 语音数据标注员  
- 语音技术爱好者  

---

## 2. 核心价值主张

**一句话描述**：将原始音频一键转换为可直接用于 TTS 训练的标准化数据集。

| 价值 | 说明 |
|------|------|
| **自动化** | 大幅减少人工处理时间 |
| **精准** | AI 辅助 + 人工校验的混合工作流 |
| **标准化** | 输出符合主流 TTS 框架的数据格式 |
| **可复用** | 支持项目管理和批量处理 |

### 3.2 功能模块详细说明

#### 模块一：音频输入管理 (M001)

| 需求项 | 描述 |
|--------|------|
| **M001-1** | 支持拖拽音频文件到指定区域上传 |
| **M001-2** | 支持多文件批量上传 |
| **M001-3** | 支持的音频格式：WAV、MP3、FLAC、M4A、OGG |
| **M001-4** | 显示上传文件列表，支持删除 / 重新排序 |
| **M001-5** | 音频元信息显示（时长、采样率、声道数） |

**用户交互流程**：拖放或选择文件 → 列表管理 → 点击「开始处理」进入下一步。

---

#### 模块二：音频预处理与格式化 (M002)

| 需求项 | 描述 |
|--------|------|
| **M002-1** | 音频响度标准化（统一到 -18 LUFS） |
| **M002-2** | 音频格式统一（采样率、声道标准化） |
| **M002-3** | 音频质量检测（静音段、截幅检测） |
| **M002-4** | 预处理进度显示 |

**预处理参数**：目标响度 -18 LUFS；真峰值限制 -1 dBTP；输出格式 WAV 48kHz 单声道。

---

#### 模块三：音频智能切分 (M003)

| 需求项 | 描述 |
|--------|------|
| **M003-1** | 基于 VAD 自动检测静音段并切分 |
| **M003-2** | 切片时长 5–15 秒可配置 |
| **M003-3** | 支持最小 / 最大切片时长参数 |
| **M003-4** | 切分后切片预览和播放 |
| **M003-5** | 支持手动调整切分边界 |
| **M003-6** | 过滤过短(&lt;3s)或过长(&gt;20s)的无效切片 |

**切分要求**：基于 VAD、优先句间停顿、避免词中切分。

---

#### 模块四：ASR 语音识别 (M004)

| 需求项 | 描述 |
|--------|------|
| **M004-1** | 对切片批量 ASR 识别 |
| **M004-2** | 支持中文普通话 |
| **M004-3** | 支持粤语 |
| **M004-4** | 显示识别进度和预计完成时间 |
| **M004-5** | 支持本地与云端 ASR 双模式 |
| **M004-6** | 识别结果缓存，避免重复识别 |
| **M004-7** | 支持 ASR 引擎切换与配置 |

**本地模式**：Whisper（base/small/medium/large），语言 zh/yue。**云端模式**：阿里云 FunASR、讯飞、百度等，需 API 配置。

---

#### 模块五：文本后处理 (M005)

| 需求项 | 描述 |
|--------|------|
| **M005-1** | 自动去除标点 |
| **M005-2** | 普通话转无调拼音（如 `ni hao`） |
| **M005-3** | 粤语转无调粤拼（如 `nei hou`） |
| **M005-4** | 语言自动检测 |
| **M005-5** | 特殊字符过滤与标准化 |

---

#### 模块六：人工检查校正 (M006)

| 需求项 | 描述 |
|--------|------|
| **M006-1** | 表格展示切片与对应文本 |
| **M006-2** | 支持文本直接编辑 |
| **M006-3** | 点击播放对应切片 |
| **M006-4** | 快捷键：播放/暂停、下一条、标记问题 |
| **M006-5** | 标记问题切片（音质差、识别错误等） |
| **M006-6** | 批量操作：删除、合并、跳过 |
| **M006-7** | 进度保存与恢复 |

**交互示意**：左侧切片列表，右侧编辑区（拼音文本、播放、跳过、标记、删除），底部上一条/下一条导航。

---

#### 模块七：FA 强制对齐 (M007)

| 需求项 | 描述 |
|--------|------|
| **M007-1** | 基于 MFA 音素级对齐 |

| **M007-5** | 显示对齐进度 |

**输出**：TextGrid（词级 + 音素级）。

#### 模块八：音频合并 (M008)

## 5. 用户故事

### 故事一：初次使用

**作为** AI 语音研究者，**我想要** 将录音快速转为训练数据集，**以便** 立即开始 TTS 训练。

**场景**：创建项目 → 拖拽约 10 个文件（约 30 分钟）→ 自动预处理、切分、ASR、转无调拼音 → 人工修正约 5 处 → 导出 .lab 格式；整体约 20 分钟。

---

### 故事二：批量处理

**作为** 数据标注负责人，**我想要** 批量处理大量音频，**以便** 高效产出训练数据。

**场景**：导入约 100 个文件（约 5 小时）→ 设置参数 → 后台自动处理（本地 Whisper）→ 分批人工校验 → 导出完整 .lab 数据集。

---

### 故事三：精细调整

**作为** 追求高质量的训练者，**我想要** 精确控制每个环节，**以便** 得到最优训练数据。

**场景**：确认 -18 LUFS 等预处理参数 → 调整切分（如 8–12 秒）→ 手动微调边界 → 本地 Whisper 识别 → Praat 精修 FA → 重新导入标注 → 导出高质量 .lab。

## 6. 非功能需求

### 6.1 性能需求

| 指标 | 目标值 |
|------|--------|
| 音频切分速度 | &gt; 实时 10 倍 |
| ASR 处理速度 | 取决于模型，支持 GPU 加速 |
| FA 对齐速度 | &gt; 实时 5 倍 |
| 界面响应时间 | &lt; 100ms |
| 内存占用 | &lt; 4GB（常规使用） |

### 6.2 兼容性需求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11、macOS 12+、Linux |
| 音频格式 | WAV、MP3、FLAC、M4A、OGG |
| 采样率 | 8kHz–48kHz |
| 声道 | 单声道/立体声（自动转单声道） |

### 6.3 可用性需求

- 清晰的处理进度与状态提示  
- 错误信息友好并给出可操作建议  
- 支持撤销/重做  
- 键盘快捷键  
- 暗色/亮色主题切换

## 7. 技术约束

### 7.1 核心技术栈

- **后端 / GUI**：Go + Fyne v2（跨平台桌面）  
- **音频处理**：Go 原生（beep、go-audio 等），FFmpeg 可选备用  
- **ASR**：Whisper（如 whisper.cpp）/ 云端 API  
- **FA 对齐**：Montreal Forced Aligner (MFA)  
- **状态存储**：JSON 文件（project.json、config.json），无数据库  

### 7.2 模型与依赖

- ASR 模型文件（约 1–3GB）  
- MFA 声学模型与词典  
- 可选：本地 GPU 加速
