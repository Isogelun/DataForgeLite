## 1. 产品概述

### 1.1 产品名称
**DataForge Lite** - AI语音训练数据生成工具

### 1.2 产品定位
一款面向AI语音合成（TTS）模型训练的数据预处理工具，能够将原始音频自动化处理为标准化的训练数据集。

### 1.3 目标用户
- AI语音研究者/开发者
- TTS模型训练工程师
- 语音数据标注员
- 语音技术爱好者

---

## 2. 核心价值主张

**一句话描述**: 将原始音频一键转换为可直接用于TTS训练的标准化数据集。

**核心价值**:
- 🚀 自动化：大幅减少人工处理时间
- 🎯 精准：AI辅助+人工校验的混合工作流
- 📊 标准化：输出符合主流TTS框架的数据格式
- 🔄 可复用：支持项目管理和批量处理

### 3.2 功能模块详细说明

#### 模块一：音频输入管理 (M001)

| 需求项 | 描述 |
|--------|------|
| **M001-1** | 支持拖拽音频文件到指定区域上传 |
| **M001-2** | 支持多文件批量上传 |
| **M001-3** | 支持的音频格式：WAV、MP3、FLAC、M4A、OGG |
| **M001-4** | 显示上传文件列表，支持删除/重新排序 |
| **M001-5** | 音频元信息显示（时长、采样率、声道数） |

**用户交互流程**:
1. 用户打开应用，看到拖放区域
2. 拖拽音频文件到区域，或点击选择文件
3. 文件列表显示，可进行管理操作
4. 点击"开始处理"进入下一步

---

#### 模块二：音频预处理与格式化 (M002)

| 需求项 | 描述 |
|--------|------|
| **M002-1** | 音频响度标准化（统一到-18 LUFS） |
| **M002-2** | 音频格式统一转换（采样率、声道标准化） |
| **M002-3** | 音频质量检测（静音段、截幅检测） |
| **M002-4** | 预处理进度显示 |

**预处理参数**:
- 目标响度：-18 LUFS（符合广播标准）
- 真峰值限制：-1 dBTP
- 输出格式：WAV 48kHz 单声道

#### 模块三：音频智能切分 (M003)

| 需求项 | 描述 |
|--------|------|
| **M003-1** | 自动检测音频静音段进行智能切分 |
| **M003-2** | 切片时长控制在5-15秒范围内 |
| **M003-3** | 支持设置最小/最大切片时长参数 |
| **M003-4** | 切分后切片预览和播放 |
| **M003-5** | 支持手动调整切分边界 |
| **M003-6** | 过滤过短(<3s)或过长(>20s)的无效切片 |

**切分算法要求**:
- 基于VAD（语音活动检测）识别静音边界
- 优先在句间停顿处切分
- 避免在词中间切分

---

#### 模块四：ASR语音识别 (M004)

| 需求项 | 描述 |
|--------|------|
| **M004-1** | 对切分后的音频切片进行批量ASR识别 |
| **M004-2** | 支持中文普通话识别 |
| **M004-3** | 支持粤语识别 |
| **M004-4** | 显示识别进度和预计完成时间 |
| **M004-5** | **支持本地和云端ASR双模式** |
| **M004-6** | 识别结果缓存，避免重复识别 |
| **M004-7** | 支持ASR引擎切换配置 |

**本地ASR模式**:
- 引擎：本地Whisper模型
- 模型大小：base/small/medium/large（可选）
- 语言：zh（普通话）、yue（粤语）
- 优点：无需网络、隐私安全、长期免费

**云端ASR模式**:
- 引擎：阿里云FunASR、讯飞API、百度API等
- 优点：识别精度高、处理速度快、无需本地GPU
- 配置：API Key管理、网络超时设置

**智能选择策略**:
- 默认优先使用本地模型
- 本地模型未配置时提示使用云端
- 支持按项目/批次选择不同模式

---

#### 模块五：文本后处理 (M005)

| 需求项 | 描述 |
|--------|------|
| **M005-1** | 自动去除标点符号 |
| **M005-2** | 普通话文本转换为拼音（不带声调） |
| **M005-3** | 粤语文本转换为粤拼（不带声调） |
| **M005-4** | 支持语言自动检测 |
| **M005-5** | 特殊字符过滤和标准化 |

**拼音转换要求**:
- 格式：**不带声调**，如 "ni hao"
- 全小写字母，空格分词
- 支持多音字根据上下文智能选择

**粤拼转换要求**:
- 格式：**不带声调**，如 "nei hou"
- 标准Jyutping，去除数字声调标记

#### 模块十：数据集导出 (M010)

| 需求项 | 描述 |
|--------|------|
| **M010-1** | 按TextGrid标注拆分音频和文本 |
| **M010-2** | 生成 `.lab` 格式标注文件（与音频同名） |
| **M010-3** | 支持多种TTS框架格式（VITS、Bert-VITS2等） |
| **M010-4** | 生成数据集元信息文件 |

**支持的输出格式**:
- **默认格式**：wav文件 + 同名.lab文件
- `.lab` 文件格式：纯文本拼音/粤拼，每行一个切片文本
- VITS格式：wav文件 + transcript.txt
- Bert-VITS2格式：wav文件 + 标注文件 + 配置文件

**输出示例**:
```
output/
├── audio_001.wav
├── audio_001.lab      # 包含: ni hao shi jie
├── audio_002.wav
├── audio_002.lab      # 包含: wo ai bei jing tian an men
└── metadata.json
```

## 5. 用户故事

### 故事一：初次使用
**作为** 一位AI语音研究者
**我想要** 将我的录音文件快速转换为训练数据集
**以便于** 能够立即开始TTS模型的训练

**场景**:
1. 打开软件，创建新项目
2. 拖拽10个录音文件（共30分钟）到上传区
3. 点击"开始处理"
4. 系统自动完成音频格式化（响度标准化-18 LUFS）、切分、ASR、拼音转换（无调）
5. 我检查并修正了5处识别错误
6. 导出为.lab格式数据集（音频+同名lab文件）
7. 整个流程用时约20分钟

---

### 故事二：批量处理
**作为** 一个数据标注团队负责人
**我想要** 批量处理大量音频文件
**以便于** 高效生成训练数据

**场景**:
1. 导入100个音频文件（约5小时）
2. 设置自动处理参数（响度标准化、切分参数）
3. 系统在后台自动处理（使用本地Whisper模型）
4. 分批进行人工校验拼音文本
5. 最终导出.lab格式完整数据集

---

### 故事三：精细调整
**作为** 一个追求完美的模型训练者
**我想要** 精确控制每个处理环节
**以便于** 获得最高质量的训练数据

**场景**:
1. 确认音频预处理参数（-18 LUFS响度标准）
2. 调整切分参数，设置最小8秒、最大12秒
3. 手动调整不满意的分段边界
4. 使用本地Whisper模型进行识别
5. 在Praat中精修FA对齐结果
6. 重新导入精修后的标注
7. 导出.lab格式高质量数据集（无调拼音）

## 6. 非功能需求

### 5.1 性能需求

| 指标 | 目标值 |
|------|--------|
| 音频切分速度 | > 实时速度的10倍 |
| ASR处理速度 | 取决于模型，支持GPU加速 |
| FA对齐速度 | > 实时速度的5倍 |
| 界面响应时间 | < 100ms |
| 内存占用 | < 4GB（常规使用） |

### 5.2 兼容性需求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11, macOS 12+, Linux |
| 音频格式 | WAV, MP3, FLAC, M4A, OGG |
| 采样率 | 支持8kHz-48kHz |
| 声道 | 单声道/立体声（自动转单声道） |

### 5.3 可用性需求

- 提供清晰的处理进度指示
- 错误信息友好且可操作建议
- 支持撤销/重做操作
- 键盘快捷键支持
- 暗黑/明亮主题切换

## 7. 技术约束

### 6.1 核心技术栈
- **后端**: Go语言（主框架）
- **前端**: Wails框架（Web技术栈）
- **音频处理**: FFmpeg, librosa
- **ASR**: Whisper/FunASR
- **FA对齐**: Montreal Forced Aligner

### 6.2 模型依赖
- ASR模型文件（约1-3GB）
- MFA声学模型和词典
- 可选：本地GPU加速支持

## 8. 附录

### 7.1 术语表
